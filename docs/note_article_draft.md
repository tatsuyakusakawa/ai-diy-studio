# Note記事 最終稿

---

**3行でわかること：ComfyUI + n8n を使ってAI画像・動画を半自動生成する仕組みを作った。ワークフローはそのまま配布する。環境構築はClaude CodeなどのCLIツールに任せればほぼ自動でできる。**

---

## 何が作れるか、作れないか（先に正直に言う）

**得意なもの**
- 食材・料理のアップ（質感がよく出る）
- 風景・城下町・建物（人物なしで安定する）
- 動物のクローズアップ
- 雰囲気重視のイメージカット

**まだ苦手なもの**
- 複数人物が登場するストーリー（腕や足が変な形になることがある）
- 複数カットで同じ人物・服装を維持すること

これはこのセットアップだけでなく、2026年現在のAI画像生成全体の課題らしい。逆に言えば、人物が出てこない映像ならかなり使える。

---

## 必要なもの

**ハード**
- M4 Mac（メモリ48GB）※Windowsでも動くはずだが未検証

**ソフト（無料）**
- Docker Desktop（手動インストール必要）
- ComfyUI
- n8n

**モデル（無料でダウンロード）**
- FLUX.2 Klein 9B（画像生成）
- LTX-2.3（動画生成）
- Gemma 3 12B GGUF（テキストエンコーダ）

課金は一切なし。モデルのダウンロードに合計50GB以上の空き容量と時間がかかる。

---

## セットアップ方法

GitHubにすべて置いてある。

👉 **https://github.com/tatsuyakusakawa/ai-diy-studio**

```
workflows/   ← n8nにインポートするだけで動くJSONファイル
prompts/     ← プロンプトの書き方ガイド
docs/        ← モデルのダウンロード先一覧、手順
CLAUDE.md    ← CLIツールへの指示書
```

**Docker Desktopとモデルのダウンロードだけ手動で済ませたら**、あとはClaude Codeなどに以下を伝えるだけ：

```
このリポジトリのCLAUDE.mdを読んで、環境をセットアップしてください
```

ComfyUIのインストール、カスタムノードの導入、n8nのDockerセットアップまで自動でやってくれる。

---

## 仕組み（半自動）

**ステップ1**：プロンプトを書いてn8nを実行 → 指定したカット数の画像が順番に自動生成される

**ステップ2**：生成された画像を確認・選別（ここだけ自分が判断する）

**ステップ3**：各カットのモーションプロンプトを書いてn8nを実行 → 動画が自動生成される

画像と動画の間に自分の目が入る構造。「全自動」ではなく「半自動」。

## 詰まったところ（正直に）

- LTX-2.3の音声生成がGGUF実装の制限で動かなかった
- n8nのループが途中で壊れていた（状態管理の設計の問題）
- モデルファイルを置く場所が決まっていて、間違えると動かない
- ComfyUIとDockerからの接続先が違う（`localhost`ではなく`host.docker.internal`）

細かい詰まりポイントは次の記事で書く。
